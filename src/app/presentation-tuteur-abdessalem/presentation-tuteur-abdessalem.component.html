Le routage dans Angular est un mécanisme qui permet de naviguer entre différentes pages (ou composants) d'une application à **page unique (SPA)** sans recharger la page entière.

Lorsqu'on change d'URL (par exemple de /home à /about), Angular intercepte ce changement et affiche dynamiquement le composant correspondant, sans recharger toute l'application.

Démonstration (1 min) : Je vais vous montrer une petite application Angular où on clique sur différents liens du menu, et on observe :
  • Le changement de l’URL dans la barre d’adresse (path)
  • Le contenu affiché qui change automatiquement
  • Différents types de navigation : par clic, par logique conditionnelle, par programmation…
Tout cela est possible grâce au module de Router d’Angular, qui permet de gérer ces comportements de manière propre et modulaire.

Utiliser un router dans Angular présente plusieurs avantages :
  • Permet de construire des applications à pages multiples (mais en réalité, c’est une seule page HTML qui évolue dynamiquement).
  • Améliore l'expérience utilisateur avec des transitions rapides.
  • Structure l'application en modules et composants bien séparés.
  • Permet de gérer facilement la navigation conditionnelle (authentification, autorisation, etc.).
  • Supporte le lazy loading pour améliorer les performances.
Sans router, chaque changement de "page" nécessiterait de recharger toute l’application – ce qui irait à l'encontre des principes SPA (Single Page Application).

Pour démarrer une nouvelle application Angular avec le système de routage intégré, deux méthodes s’offrent à vous :

✔️ Méthode 1 – Avec le flag --routing

Commande directe pour générer l’application avec le routage : 

ng new angular-routing-demo --routing

Cela crée automatiquement :

  • Le fichier app-routing.module.ts
  • L’import du RouterModule dans AppModule

Ensuite :

cd angular-routing-demo
ng serve
✔️ Méthode 2 – En mode interactif (visible sur la capture)

Lancez simplement :

ng new nom-de-ton-app

Angular vous posera des questions :

  • Which stylesheet format would you like to use? → Choisissez CSS, SCSS, etc.

Exemple : Création en mode interactif avec Angular CLI.

Vous pouvez ensuite créer vos composants et configurer vos routes dans app-routing.module.ts !

Dans le fichier index.html d’un projet Angular, on trouve souvent la ligne suivante : 
<base href="/">
Cette balise a un rôle crucial dans le fonctionnement de l’application :
  • 📍 Elle définit la racine de toutes les URLs utilisées dans l’application.
  • 📦 Elle permet au router Angular de savoir comment gérer les routes comme /accueil ou /contact.
  • ❌ Sans elle, la navigation dans l’application risque de générer des erreurs 404 lors de l’actualisation des pages.
Exemple : Si on a une image à assets/logo.png :
  • Avec <base href="/"> ➜ URL correcte : /assets/logo.png
  • Avec <base href="/mon-app/"> ➜ l’image est cherchée à : /mon-app/assets/logo.png

C’est donc une configuration essentielle pour que le système de routage et les chemins relatifs fonctionnent correctement dans Angular.

Configuration des routes

1. Pas à pas : définition d’une route

Toutes vos routes se trouvent dans app-routing.module.ts :

Explications :

  • path : segment de l’URL à matcher.
  • component : composant à afficher.
  • redirectTo : URL de redirection.
  • pathMatch: 'full' : match exact de l’URL.
  • RouterModule.forRoot(routes) : méthode qui configure le routeur au niveau racine.
L'étape finale consiste à ajouter la balise < router-outlet> < /router-outlet> dans le fichier app.component.html , car c'est à cet emplacement que les composants correspondant aux routes définies seront affichés. Cette balise sert de point d'ancrage pour la navigation, permettant ainsi à l'application de charger dynamiquement les vues en fonction des routes sélectionnées par l'utilisateur.

2. RouterModule – forRoot vs forChild

Le module RouterModule configure le système de routing :

  • forRoot(routes) : à utiliser dans AppRoutingModule (module racine).
  • forChild(routes) : à utiliser dans les modules enfants (lazy-loaded).

C'est quoi ActivatedRoute ?
ActivatedRoute est un service Angular qui donne accès aux informations de la route actuelle. La différence essentielle réside dans la réactivité : La méthode params.subscribe utilise un Observable qui met à jour dynamiquement l'ID si la route change sans recharger le composant (idéal pour les SPA), tandis que snapshot récupère la valeur une seule fois à l'initialisation (adapté quand le composant est rechargé à chaque navigation). La première nécessite une gestion des abonnements, la seconde est plus simple mais moins flexible.

7. RouterLinkActive

Applique une classe au lien actif : 

<nav>
                  <a routerLink="/home" routerLinkActive="active">Accueil</a>
                  <a routerLink="/about" routerLinkActive="active">À propos</a>
                </nav>

                <router-outlet></router-outlet>

Nous allons créer un composant Navbar qui contient des liens de navigation vers différentes pages de notre application : la page d'accueil (/home), la page d'inscription (/signin) et la liste des produits (/list-produits). Pour savoir quelle route est actuellement active et appliquer un style visuel distinct (comme une couleur rouge), nous utilisons la directive Angular routerLinkActive="active" combinée à [routerLinkActiveOptions]=" exact: true ". Cette directive ajoute dynamiquement une classe CSS (active) au lien correspondant à la route en cours, ce qui nous permet de le styliser facilement.

Dans une application Angular, il est souvent utile d’organiser les composants selon une hiérarchie, surtout lorsqu’un composant "parent" sert de conteneur à plusieurs composants "enfants". C’est exactement ce qu’on appelle les routes enfants ou child routes. Imaginons qu’on a une section /produits dans notre application. Cette section peut afficher différentes vues selon le contexte :
  • /produits/liste → Affiche la liste des produits
  • /produits/ajouter → Formulaire pour ajouter un nouveau produit
  • /produits/:id → Détails d’un produit spécifique Dans ce cas, ProduitsComponent est le parent qui contient un
  • < router-outlet> pour afficher les vues enfants.
Configuration Basique :
  • ✅ Meilleure organisation des composants
  • ✅ Facilité de navigation contextuelle (garder le même layout ou header)
  • ✅ Contrôle plus précis sur les autorisations ou les guards par sous-route

Histoire animée : Quand une application Angular devient grande, charger tous les modules dès le début peut ralentir le démarrage. C’est là que le lazy loading devient très utile.
❓ C’est quoi le lazy loading ?

Le lazy loading permet de charger un module uniquement quand il est nécessaire, c’est-à-dire au moment où l’utilisateur accède à une certaine route.
